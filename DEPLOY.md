# Deploy Guide — Restocked

Goal: get the app live on a public URL with a database and the nightly jobs.
Estimated time: 1–2 hours. Everything here is a step only you can do (accounts,
secrets, deploy).

## 0. Prerequisites
- A GitHub account (push this folder to a private repo).
- A Shopify Partner account + a development store (see YOUR-NEXT-STEPS.md).
- A Render account (render.com) — free tier is fine to start.

## 1. Push to GitHub
From this folder:
    git init && git add . && git commit -m "Restocked app"
    git branch -M main
    git remote add origin <your-repo-url>
    git push -u origin main
(`.gitignore` already excludes `.env` and caches.)

## 2. Create the services on Render
Option A — Blueprint (easiest): in Render, New → Blueprint, point it at your repo.
It reads `render.yaml` and creates: the web service, a Postgres database, and two
cron jobs (nightly refresh + weekly digest).

Option B — manual: New → Web Service (build `pip install -r requirements.txt`,
start `uvicorn app.main:app --host 0.0.0.0 --port $PORT`), add a Postgres
instance, then add the two Cron Jobs with commands `python -m jobs.refresh_all`
and `python -m jobs.send_digests`.

## 3. Set environment variables (Render dashboard → each service → Environment)
- SHOPIFY_API_KEY / SHOPIFY_API_SECRET — from your Partner app (step in YOUR-NEXT-STEPS).
- SHOPIFY_APP_HANDLE — the App Store app handle used in the Shopify App Pricing
  URL. Current value: `restocked-7`.
- APP_URL — your Render web URL, e.g. https://restocked.onrender.com
- DATABASE_URL — auto-filled if you used the Blueprint; else paste the Postgres URL.
- APP_SECRET, CRON_SECRET — auto-generated by the Blueprint, or set any long random strings.
- Email (optional, for the digest): SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, FROM_EMAIL.
  Use any SMTP provider (e.g. a transactional email service). If unset, digests are skipped safely.

## 4. Point Shopify at your URL
In the Partner dashboard → your app → App setup:
- App URL: https://<your-app>/
- Allowed redirection URL: https://<your-app>/auth/callback
- (Embedded app: ON)

In Partner Dashboard → Pricing:
- Confirm Starter/Growth/Pro are configured with Shopify App Pricing.
- Set each plan's welcome/redirect link to `/billing/callback` or
  `https://<your-app>/billing/callback`.

## 5. Configure the mandatory webhooks (for review)
In App setup → Compliance webhooks, set all three to:
- https://<your-app>/webhooks/customers/data_request
- https://<your-app>/webhooks/customers/redact
- https://<your-app>/webhooks/shop/redact
Add the app-uninstalled webhook (Notifications): https://<your-app>/webhooks/app/uninstalled

## 6. Smoke test on your dev store
- Visit https://<your-app>/install?shop=<your-dev-store>.myshopify.com
- Approve install → you land on the embedded dashboard.
- Add a few products with size/color variants and some test orders, then hit
  "Refresh forecast" and confirm recommendations appear.

## 7. Jobs
- The cron services run automatically (refresh daily 05:00 UTC, digest Mondays 13:00 UTC).
- To trigger manually you can also POST the HTTP endpoints with the secret header:
    curl -X POST https://<your-app>/tasks/refresh-all -H "X-Cron-Secret: <CRON_SECRET>"
    curl -X POST https://<your-app>/tasks/digest      -H "X-Cron-Secret: <CRON_SECRET>"

## Run locally first (optional but recommended)
    pip install -r requirements.txt
    cp .env.example .env   # fill API key/secret + APP_SECRET
    DEMO=1 uvicorn app.main:app --reload
    # open http://localhost:8000/?shop=demo-store.myshopify.com
